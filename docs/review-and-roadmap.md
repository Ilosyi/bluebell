# 项目 Review 与后续规划

## 当前主要不足

1. 帖子与社区查询曾经存在明显的 N+1 模式，列表与详情聚合成本高。
2. 不存在的帖子/社区曾统一返回“服务繁忙”，错误语义不够清晰。
3. 列表分页信息原先不足，前端只能靠当前页条数猜测是否还有下一页。
4. 前端论坛数据流分散在页面里，筛选、分页、刷新和错误处理耦合在 `Home.vue`。
5. 登录回跳逻辑不统一，发帖和投票的未登录体验不一致。

## 本轮改进方向

1. 后端增加更明确的错误码：
   - `CodePostNotFound`
   - `CodeCommunityNotFound`
2. 帖子详情和帖子列表聚合查询下沉到 DAO，减少 Logic 层逐条查库。
3. 帖子列表接口返回分页元数据，前端分页基于真实 `total / total_pages / has_more`。
4. 前端抽出 `useForumList` composable，统一首页的数据与分页逻辑。
5. 未登录投票时带上 `redirect` 回跳参数，登录成功后回到原帖。

## 建议的后续开发路线

### 1. 接口与数据层

1. 给 `/posts2` 增加更稳定的分页契约文档。
2. 为帖子详情、社区详情增加“资源不存在”的统一领域错误映射。
3. 继续把作者、社区、帖子聚合查询往 DAO / Repository 层收敛。

### 2. 前端结构

1. 继续把帖子详情、发帖页也抽成 composable。
2. 给论坛配置增加更多可配项，例如首页标题、热议模块标题、默认排序方式。
3. 补前端 E2E 测试，覆盖登录、列表、详情、投票、发帖主流程。

### 3. 产品功能

1. 评论 / 回复系统。
2. 社区独立详情页与社区主题列表页。
3. 搜索、标签、置顶、推荐主题。
4. 我的帖子、编辑帖子、删除帖子、草稿箱。
5. 用户资料页、活跃记录、勋章或等级系统。

### 4. 运营与维护

1. 公告位、热议榜、推荐内容模块配置化。
2. 开发环境一键启动与种子数据脚本。
3. 更清晰的日志与错误追踪，减少“服务繁忙”式模糊反馈。
